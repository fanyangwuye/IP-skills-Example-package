# 互动影游模板

按需读取。用于把 IP 改成互动影游、可玩 Demo、选择分支视频。

## 必备结构

互动影游输出必须分三层：**游戏玩法层 → 剧情反馈层 → 选择问题层**。

```markdown

### 玩家身份
- 玩家是谁：
- 为什么进入这个世界：
- 初始限制：
- 初始目标：

### 核心循环
探索 → 选择 → 获得线索/能力/关系变化 → 遭遇后果 → 进入下一节点

### 系统设计
- 阵营好感：
- 信任值：
- 能力成长：
- 关键道具：
- 线索收集：
- 失败条件：

## 2. 剧情反馈层
| 玩家选择 | 即时反馈 | 关系变化 | 线索/资源变化 | 后续风险 |
|---|---|---|---|---|

## 3. 选择问题层
### 选择节点 1：
问题：你现在要怎么做？
A. 行动选项A
- 可见后果：
- 隐藏影响：
B. 行动选项B
- 可见后果：
- 隐藏影响：
C. 行动选项C（可选）
- 可见后果：
- 隐藏影响：

## 分支路线
| 路线 | 触发选择 | 主要冲突 | 结局倾向 |
|---|---|---|---|

## Demo视频结构
- 0-15s：世界入口
- 15-30s：第一次选择问题
- 30-60s：选择后剧情反馈
- 60-90s：系统/阵营/能力展示
- 90-120s：第二次选择问题
- 120-180s：后果升级与悬念召回
```

## 互动影游原则

- 玩家最好是新角色，不要强行操控原作主角。
- 先设计玩法骨架，再写剧情反馈，再写选择问题。
- 选择要产生可见后果，不要假互动。
- 每个选择节点给 2-3 个选项，禁止十几个选项轰炸用户。
- 每个选项必须写“可见后果”，可选写“隐藏影响”。
- 每个分支都要服务 IP 核心冲突。
- Demo 优先做 3 分钟，别一上来设计 30 小时大作，醒醒。
