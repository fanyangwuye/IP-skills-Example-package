## 文案 Skill 当前能力

`ip-copy-skill` 已支持：

- `check_license`：授权检查
- `update_adaptation_state`：互动式二创状态沉淀
- `build_adaptation_scene_cards`：从二创状态生成场景卡
- `build_script_draft`：从场景卡或状态生成结构化短剧脚本草稿
- `polish_script_draft`：对白短句化、冲突强化、保留原对白
- `build_blueprint`：从 scene cards 生成剪辑蓝图
- `build_character_handoff`：组织图片 handoff
- `build_ip_asset_pack`：从文案提取多角色、道具、场景，生成 `ip_asset_pack`

重要实现文件：

- `skills/ip-copy-skill/scripts/copy_skill.py`
- `skills/ip-copy-skill/scripts/blueprint_validate.py`
- `skills/ip-copy-skill/scripts/license_gate.py`

关键规则：

- `build_ip_asset_pack` 不只分析主角。
- 显式传入的 `characters` 会全部保留。
- 只有 `source_text` 时，会尝试抽取多角色、称谓角色、怪物角色、员工、盟友、对手、场景驱动角色。
- 道具按角色所在句子附近上下文绑定。
- 场景默认按 720 全景环境参考输出。

## 音乐 Skill 当前能力

`ip-music-skill` 已支持：

- `build_music_handoff`：从 `polished_script`、`script_draft`、`scene_cards` 或 `blueprint` 生成主题曲和场景 BGM 任务
- `generate_music`：文生音乐
- `add_instrumental`：给上传音频配伴奏
- `add_vocals`：给伴奏加人声
- `cover_audio`：上传音频翻唱 / 换风格 / 旋律重制
- `extend_music`：续写已有生成曲
- `replace_section`：替换已有生成曲片段
- `stem_split`：对 PoYo/Suno 生成曲拆分多轨
- `separate_vocals`：对 PoYo/Suno 生成曲做人声/伴奏分离
- `upload_separate_vocals`：对用户上传的外部音频做拆轨/分离

重要实现文件：

- `skills/ip-music-skill/SKILL.md`
- `skills/ip-music-skill/scripts/music_skill.py`
- `skills/ip-music-skill/scripts/poyo_music_client.py`
- `skills/ip-music-skill/scripts/config.py`
- `skills/ip-music-skill/references/workflows.md`

关键规则：

- 离线 `build_music_handoff` 不耗积分。
- live 音乐调用需要 `MUSIC_API_KEY` 或兼容别名 `POYO_API_KEY`。
- 音乐查询端点是 `/api/generate/detail/music`，不要套用图片状态端点。
- `stem_split`、`separate_vocals`、`replace_section` 等已有曲操作需要上游 `task_id` 和 `audio_id`。
- `upload_separate_vocals` 用于用户上传的外部音频，传 `audio_path` 或 `audio_url`。

## Live 图片生成配置

不要把 key 写入仓库。临时 PowerShell 环境变量：

```powershell
$env:IMAGE_PROVIDER="poyo"
$env:IMAGE_API_KEY="你的key"
$env:IMAGE_API_BASE="https://api.poyo.ai"
$env:IMAGE_GEN_MODEL="gpt-image-2"
$env:IMAGE_EDIT_MODEL="gpt-image-2-edit"
$env:IMAGE_OUTPUT_ROOT="E:\Plans for 2026\ip-skills\outputs"
```

优先小规模验证：

1. 文案生成 `ip_asset_pack`
2. 只取第一个角色任务
3. 生成 1 张角色设定图
4. 检查中文标签、道具绑定、画质、背景干净度

不要一开始就生成整套资产包，避免浪费额度。

## Live 音乐生成配置

不要把 key 写入仓库。临时 PowerShell 环境变量：

```powershell
$env:MUSIC_PROVIDER="poyo"
$env:MUSIC_API_KEY="你的key"
$env:MUSIC_API_BASE="https://api.poyo.ai"
$env:MUSIC_OUTPUT_ROOT="E:\Plans for 2026\ip-skills\outputs"
$env:MUSIC_DEFAULT_MODEL_VERSION="V5"
```

优先小规模验证：

1. 文案生成 `polished_script` 或 `blueprint`
2. 音乐 Skill 生成 `music_handoff`
3. 只取主题曲或第一个 BGM 任务生成一首短音频
4. 记录返回的 `task_id` 和 `audio_id`
5. 再验证一次 `stem_split` 或 `separate_vocals`

## 开发原则

- 不把 API key 写入文件。
- 不提交 `outputs/` 生成图。
- 做完一个功能点就运行相关测试并提交。
- 不要把确定性结构层改成不可测试的自由生成。
- 外部模型能力应作为可选增强，不能破坏无 key 离线测试。
