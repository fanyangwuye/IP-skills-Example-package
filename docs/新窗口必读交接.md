## 3. 视频生成固定规则

### 3.1 模型策略

- PoYo live 视频默认 `seedance-2`。
- 不默认用 `seedance-2-fast`。
- 只有用户明确说要 fast，并且任务设置 `allow_fast_model=true`，才允许 fast。
- 不因为 fast 便宜或快就擅自切模型。

### 3.2 参考图策略

如果项目设置：

```json
{"reference_policy": "all_purpose_reference"}
```

则 live character clip 必须使用：

```text
reference_image_urls
```

禁止自动改成：

- `image_urls`
- 首帧 I2V
- 尾帧 I2V
- 上一段尾帧当首帧
- keyframe I2V

单帧截取只服务于多段视频衔接参考，可用于继承光色、服装、动作余势、屏幕方向、角色状态和情绪残留。它不是角色身份锁，也不是默认首帧，更不是全能参考替代品。

### 3.3 故事板策略

用户说“故事板”“分镜故事板”“线稿漫画故事板”时，默认理解为：

- 多格 `[Image #1] [Image #2] ...` 式 storyboard board
- 用于锁构图、景别、机位、动作相位、屏幕方向、剪辑顺序
- 不是单张最终海报
- 不是最终视频风格
- 不要把线稿表格、边框、文字、箭头、手写标记带进最终视频

故事板一旦进入 production 阶段，就是执行蓝图。每个 live clip 必须带：

```text
storyboard_execution_map
```

视频 shot 顺序必须匹配 storyboard `shot_ids`。

不要为了 15 秒生成方便而擅自删除、合并、改顺序、改动作。做不到就拆成更短 clip，或先征得用户确认修改故事板。

### 3.4 15 秒 clip 策略

- 优先 clip 级生成，不要把一集拆成过多小段。
- 但 15 秒不是硬塞长镜头。一个 15 秒 clip 可以包含多个 storyboard shots。
- 每个 clip 必须有叙事动作，不允许为了填时长让角色原地跑、原地转身、无因果移动。
- 如果 15 秒无法准确执行故事板，就拆短，不许乱改故事。

### 3.5 视频衔接策略

多段视频衔接处可以使用：

- `hard_first_frame`
- `previous_reference_reframe`
- `bridge_cutaway`
- `sound_bridge_or_hard_cut`

常用衔接方式：

- 前一段尾帧或其它 reviewed frame 只作为连续性参考
- 下一段可以重构为远景、中景、近景、特写、背影、反打、道具插入、手部局部、空镜
- 空镜/道具/手部/门窗/影子/地面反光/风尘/光线变化可以作为桥接 clip

不要简单理解成“上一段尾帧 = 下一段首帧”。

## 4. Prompt Packet 固定架构

clip 级视频提示词必须保留 Prompt Packet V1：

- `Global Context`
- `Internal Story Facts`
- `Reference Bindings`
- `Spatial Blocking`
- `15s Timeline`
- `Continuation Contract`
- `Platform-Safe Surface Wording`
- `Execution Constraints`

当前实现已经分为：

- `clip_prompt`：完整审计包
- `i2v_prompt`：I2V provider surface packet
- `seedance_prompt`：Seedance provider surface packet
- `t2v_prompt`：T2V preview surface packet

不要把四种 prompt 再合并成同一段。

## 6. 仍待补的高优先级事项

1. Prompt 长度预算
   - 为 `seedance / i2v / t2v` 设置长度上限和裁剪策略
   - 测试 provider prompt 不超过预算

2. Storyboard draft / production 模式
   - `draft`：允许提出拆分、合并、调整建议，但不能静默改
   - `production`：严格锁死故事板顺序和内容

3. 角色代词/称号上下文
   - 增强“他、她、那人、老板、侍者、师尊、魔尊”等上下文继承
   - 支持用户传入角色别名表

4. clip 级质量报告
   - 角色是否明确
   - 场景是否明确
   - 故事板映射是否完整
   - 空间轴线/屏幕方向/眼线是否完整
   - prompt 是否过长
   - negative 是否过长

5. 文案链路重构
   - 这是下一个大头
   - 目标是 LLM 创作引擎 + FormatAdapter + 多集结构 + 竖屏短剧 V1 adapter + dry_run/live 边界
