## 3. 已完成：视频生成固定规则

### 3.1 模型策略

- PoYo live 视频默认 `seedance-2`。
- 不默认用 `seedance-2-fast`。
- 只有用户明确说要 fast，并且任务设置 `allow_fast_model=true`，才允许 fast。
- 不因为 fast 便宜或快就擅自切模型。

### 3.2 参考图策略

如果项目设置：

```json
{"reference_policy": "all_purpose_reference"}
```

则 live character clip 必须使用：

```text
reference_image_urls
```

禁止自动改成：

- `image_urls`
- 首帧 I2V
- 尾帧 I2V
- 上一段尾帧当首帧
- keyframe I2V

单帧截取只服务于多段视频衔接参考，可用于继承光色、服装、动作余势、屏幕方向、角色状态和情绪残留。它不是角色身份锁，也不是默认首帧，更不是全能参考替代品。

### 3.3 故事板策略

用户说“故事板”“分镜故事板”“线稿漫画故事板”时，默认理解为：

- 多格 `[Image #1] [Image #2] ...` 式 storyboard board
- 用于锁构图、景别、机位、动作相位、屏幕方向、剪辑顺序
- 不是单张最终海报
- 不是最终视频风格
- 不要把线稿表格、边框、文字、箭头、手写标记带进最终视频

故事板一旦进入 production 阶段，就是执行蓝图。每个 live clip 必须带：

```text
storyboard_execution_map
```

视频 shot 顺序必须匹配 storyboard `shot_ids`。

不要为了 15 秒生成方便而擅自删除、合并、改顺序、改动作。做不到就拆成更短 clip，或先征得用户确认修改故事板。

### 3.4 15 秒 clip 策略

- 优先 clip 级生成，不要把一集拆成过多小段。
- 但 15 秒不是硬塞长镜头。一个 15 秒 clip 可以包含多个 storyboard shots。
- 每个 clip 必须有叙事动作，不允许为了填时长让角色原地跑、原地转身、无因果移动。
- 如果 15 秒无法准确执行故事板，就拆短，不许乱改故事。

### 3.5 视频衔接策略

多段视频衔接处可以使用：

- `hard_first_frame`
- `previous_reference_reframe`
- `bridge_cutaway`
- `sound_bridge_or_hard_cut`

常用衔接方式：

- 前一段尾帧或其它 reviewed frame 只作为连续性参考
- 下一段可以重构为远景、中景、近景、特写、背影、反打、道具插入、手部局部、空镜
- 空镜/道具/手部/门窗/影子/地面反光/风尘/光线变化可以作为桥接 clip

不要简单理解成“上一段尾帧 = 下一段首帧”。

## 7. 未完成：不要误判为已完成

以下事项目前没有真正完成，不要对用户说已经具备成熟能力：

1. `ip-copy-skill` 有 CreativeEngine prompt pack、provider request dry-run 和 provider boundary v1，但没有真实 live LLM provider 网络调用。
2. `live_llm_engine.py` 目前只构建 provider request 和 provider boundary，不会编剧、解说、对白、场景卡创作，也不会发网络请求。
3. `build_viral_explainer_script` 仍偏模板 scaffold，不能当最终爆款解说成稿。
4. fallback 路径的 `build_adaptation_scene_cards` / `build_script_draft` / `polish_script_draft` 仍是结构脚手架，不是专业编剧输出。
5. FormatAdapter 目前已有 `vertical_short_drama` V1、`overseas_short_drama` V1、`feature_film` V1、`long_series` V1、`murder_mystery` V1 和 `interactive_film_game` V1；计划内多格式 adapter V1 已补齐。
6. 多集/多幕/分支树创作结构还没完成。
7. 视频 `director_plan` 目前支持上游 override，但默认仍是确定性 fallback，不是真正 LLM 导演理解。
8. 英文 surface wording 已经不混中文原文，但目前是 glossary/visible-anchor 适配，不是完整语义级专业翻译。
9. 不能保证一键批量生成完整集数的视频质量。当前应先做单 clip / 少量 clip 受控测试。
10. 不能默认 live 生成图片、视频、音乐或 LLM 文案，必须明确确认。
