# IP Skills 使用教程草案

本文用于给新用户或协作 agent 安装、配置、测试 `ip-skills`。建议先按本文跑通离线测试，再配置 API key 做 live 生图、音乐和视频小样。

## 1. 项目说明

`ip-skills` 是一组面向 agent 的 IP 创作工作流 Skill，不是网页产品。它提供确定性结构、任务 JSON、提示词和 provider 请求层，让 Codex、Claude、OpenClaw 等 agent 可以调用本地脚本完成：

- 文案规划和短剧脚本结构化
- 角色、场景、故事板等图像任务生成
- 音乐任务规划、生成、翻唱、拆轨
- 视频连续性圣经、镜头/clip 计划、故事板、图生视频请求、连续 clip 生成

生成产物默认写入 `outputs/`，不会被 Git 上传。

## 2. 环境要求

推荐环境：

- Python：`3.10` 或更高版本
- pip：随 Python 安装
- Git：用于拉取和更新仓库
- ffmpeg / ffprobe：推荐安装，音乐本地音频代理、视频抽帧、后续拼接都需要
- Windows PowerShell / macOS Terminal / Linux shell 均可

Python 依赖：

```bash
requests>=2.32.0
Pillow>=10.0.0
```

安装依赖：

```bash
python -m pip install -r requirements.txt
```

检查环境：

```bash
python scripts/setup_doctor.py
```

## 3. 获取仓库

如果仓库是 GitHub private repo，用户需要先有访问权限。

```bash
git clone <repo-url>
cd ip-skills
python -m pip install -r requirements.txt
python scripts/setup_doctor.py
```

如果用户不会终端，可以让有本地命令权限的 agent 执行以上步骤。

## 5. API Key 配置

不要把 key 写入仓库。推荐用当前 shell 的临时环境变量。

### 通用 PoYo Key

如果同一个 PoYo key 同时用于图片、音乐和视频，可以设置：

```powershell
$env:POYO_API_KEY="你的key"
```

也可以按模块分别设置。

### 图片

```powershell
$env:IMAGE_PROVIDER="poyo"
$env:IMAGE_API_KEY="你的key"
$env:IMAGE_API_BASE="https://api.poyo.ai"
$env:IMAGE_GEN_MODEL="gpt-image-2"
$env:IMAGE_EDIT_MODEL="gpt-image-2-edit"
$env:IMAGE_OUTPUT_ROOT="E:\Plans for 2026\ip-skills\outputs"
```

### 音乐

```powershell
$env:MUSIC_PROVIDER="poyo"
$env:MUSIC_API_KEY="你的key"
$env:MUSIC_API_BASE="https://api.poyo.ai"
$env:MUSIC_OUTPUT_ROOT="E:\Plans for 2026\ip-skills\outputs"
$env:MUSIC_DEFAULT_MODEL_VERSION="V5"
```

本地 WAV/MP3 上传会自动封装黑屏 MP4 代理，需 ffmpeg。找不到时可指定：

```powershell
$env:MUSIC_FFMPEG_BIN="E:\path\to\ffmpeg.exe"
$env:MUSIC_FFPROBE_BIN="E:\path\to\ffprobe.exe"
```

### 视频

离线 dry-run：

```powershell
$env:VIDEO_PROVIDER="offline"
$env:VIDEO_OUTPUT_ROOT="E:\Plans for 2026\ip-skills\outputs"
$env:VIDEO_DEFAULT_ASPECT_RATIO="9:16"
$env:VIDEO_DEFAULT_RESOLUTION="480p"
```

PoYo video live：

```powershell
$env:VIDEO_PROVIDER="poyo_video"
$env:VIDEO_API_KEY="你的key"
$env:VIDEO_API_BASE="https://api.poyo.ai"
$env:VIDEO_OUTPUT_ROOT="E:\Plans for 2026\ip-skills\outputs"
$env:VIDEO_DEFAULT_RESOLUTION="480p"
$env:VIDEO_POLL_INTERVAL_SEC="4"
$env:VIDEO_POLL_TIMEOUT_SEC="900"
```

## 9. 视频 Skill 能做什么

位置：

```text
skills/ip-video-skill
```

主要能力：

- `build_continuity_bible`：角色/服装/道具/场景/光影一致性圣经
- `build_shot_plan`：镜头计划
- `build_clip_plan`：把镜头聚合成 5-15 秒 clip
- `storyboard_image_tasks`：生成专业分镜板/拍摄表图片任务
- `storyboard_panel_refs`：从分镜板自动裁首帧/中段/尾帧参考图
- `build_i2v_prompts`：图生视频提示词
- `build_t2v_prompts`：文生视频提示词
- `seedance_prompts`：Seedance/即梦类中文提示词
- `prepare_video_generation`：生成 provider 请求，不调用 API
- `run_video_generation`：生成单 clip
- `run_video_sequence`：多 clip 连续生成
- `build_edit_decision_list`：生成 EDL

### 视频连续性策略

当前支持两种续接模式：

#### `hard_first_frame`

上一段尾帧作为下一段首帧。

适合：

- 动作直接接上
- 一镜到底感
- 不能跳切的武戏/追逐/肢体动作

优点：首帧一帧级衔接最稳。  
缺点：构图被上一帧锁死，后续可能出现色调跳。

#### `reference_reframe`

上一段尾帧只作为连续性参考，不作为下一段首帧。

适合：

- 正常短剧剪辑
- 下一镜要换机位、换景别
- 希望减少硬接带来的色调/构图束缚

优点：更像真实剪辑，可换角度。  
缺点：不是一帧级无缝接。

默认建议：

```json
"continuation_mode": "reference_reframe"
```

必须硬接时再用：

```json
"continuation_mode": "hard_first_frame"
```

### 视频声音策略

最终流程默认：

```json
"generate_audio": true
```

允许视频模型生成：

- 雨声
- 风声
- 脚步
- 衣料摩擦
- 呼吸
- 门响
- 道具轻响
- 环境声/拟音

禁止视频模型生成：

- 背景音乐
- 歌曲
- 音乐铺底
- 画面字幕
- 标题卡
- 伪文字
- 自动台词

## 10. 推荐完整测试案例

本案例用于从文案到视频连续性跑通一条最小链路。

### 案例设定

题材：民国诡异短剧  
场景：雨夜巡捕房值班房  
角色：

- 陈渊：年轻巡捕，刚惊醒
- 赵德柱：年长巡捕，坐在桌前

目标：

1. 生成文案结构
2. 生成角色图和场景图
3. 生成视频 handoff
4. 生成专业分镜板
5. 生成第一段 5 秒视频
6. 抽第一段尾帧
7. 用 `reference_reframe` 生成第二段 5 秒视频
8. 检查人物、场景、色彩、声音、衔接是否稳定

### Step 1：离线生成视频 handoff

可使用现有示例：

```powershell
cd "E:\Plans for 2026\ip-skills"
python skills/ip-video-skill/scripts/video_skill.py --task skills/ip-video-skill/assets/example_build_video_handoff_task.json
```

真实项目中，建议从 `ip-copy-skill` 先生成 `script_draft`、`blueprint`、`ip_asset_pack`，再交给视频 Skill。

### Step 2：生成角色图和场景图

用图片 Skill 的 `ip_asset_pack` 或单独任务生成：

- `character_design_sheet`
- `video_scene_reference`

先只生成一个角色和一个场景，避免浪费额度。

### Step 3：生成专业故事板

从 `video_handoff.storyboard_image_tasks[0]` 取任务，交给图片 Skill 生成。

期望结果：

- 不是普通漫画草稿
- 是专业 storyboard board 或 shot table
- 首格对应视频首帧
- 中段/尾帧保持同一空间、同一光源、同一屏幕方向
- 标签在面板外，不压进画面主体

### Step 4：准备视频请求 dry-run

先不花额度，检查请求：

```json
{
  "mode": "prepare_video_generation",
  "provider": "offline",
  "video_handoff": "VIDEO_HANDOFF_JSON",
  "clip_index": 1,
  "storyboard_image_path": "生成的storyboard图路径",
  "reference_image_urls": [
    {"path": "角色设定图.jpg", "role": "character_design_sheet"},
    {"path": "正常场景参考图.jpg", "role": "video_scene_reference"}
  ],
  "output_dir": "outputs/test"
}
```

检查：

- `reference_image_urls` 是否包含角色图、场景图、storyboard panel refs
- prompt 是否禁止复制故事板线稿/表格/文字
- prompt 是否只保留环境声/拟音，禁止背景音乐

### Step 5：生成第一段 live 视频

确认请求没问题后：

```json
{
  "mode": "run_video_generation",
  "provider": "poyo_video",
  "dry_run": false,
  "download": true,
  "clip_index": 1,
  "generation_unit": "clip",
  "model": "seedance-2-fast",
  "duration_sec": 5,
  "resolution": "480p",
  "generate_audio": true
}
```

### Step 6：生成第二段续接视频

推荐先测 `reference_reframe`：

```json
{
  "mode": "run_video_sequence",
  "provider": "poyo_video",
  "dry_run": false,
  "download": true,
  "start_clip_index": 2,
  "max_clips": 1,
  "continuation_mode": "reference_reframe",
  "initial_previous_clip_end_frame": {
    "path": "clip_001_last_frame.jpg",
    "role": "previous_clip_end_frame",
    "source_clip_id": "clip_001"
  },
  "generate_audio": true
}
```

如果需要一帧级硬接，改为：

```json
"continuation_mode": "hard_first_frame"
```

### Step 7：检查结果

检查项：

- 第二段是否有音轨
- 是否只有环境声/拟音，没有背景音乐
- 角色脸是否漂移
- 服装是否变色
- 场景是否重置
- 门窗、桌椅、灯光方向是否连续
- 色彩、曝光、白平衡是否跳
- `reference_reframe` 是否像正常剪辑切换，而不是死复制上一帧
- `hard_first_frame` 是否一帧级接上

## 13. 常见问题

### Q：别人 clone 后能直接用吗？

能使用离线结构和测试，但 live 生图/音乐/视频需要对方配置自己的 API key。

### Q：agent 能自动安装吗？

能辅助安装 Python 依赖和运行测试，但 Python 本体、ffmpeg、API key、agent skill 注册权限可能需要用户确认。

### Q：为什么视频不要生成背景音乐？

视频模型生成背景音乐不可控，容易影响后期剪辑。BGM 和歌曲应该走音乐 Skill 和 EDL。

### Q：为什么故事板不直接整张喂给视频模型？

整张故事板有表格、文字、线稿和面板边框，容易污染视频。当前流程会自动裁出首帧/中段/尾帧 panel，只作为构图参考，并明确禁止复制线稿和文字。

### Q：多段视频怎么保持一致？

用 `run_video_sequence`。每段生成后自动抽尾帧，再按 `continuation_mode` 传给下一段：

- `hard_first_frame`：尾帧作为下一段首帧
- `reference_reframe`：尾帧作为连续性参考，允许换机位
