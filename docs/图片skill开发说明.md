# 图片 Skill 开发说明

## 目标

交付一个面向 agent 的图片 skill，不做界面层。

当前能力范围：

1. 文生图
2. 多宫格拆分
3. 拆分后逐张图生图增强
4. 通过 JSON 任务文件或 agent 调用入口执行
5. 支持小说/文案驱动的人物与资产创建
6. 支持单图的互动式逐轮细化
7. 支持基于已有单图继续细化，而不是每次重新首图生成
8. 支持基于同一角色设定一次生成多种角色资产图
9. 支持从文案资产清单生成多角色设计图、角色道具、场景全景图

## 提供商设计

主流程固定，图片提供商可替换。

当前首个接入提供商：

- `PoYo AI`

未来其他渠道接入原则：

- 优先只改环境变量
- 若协议不同，则新增 provider 适配层，不改主流程

## 用户侧配置目标

最终使用者不改代码，只需要：

1. 配置 API Key
2. 必要时覆盖 `base_url`
3. 必要时覆盖模型名
4. 按需提供任务 JSON

## 创作层设计

图片 skill 不是只有一条死 prompt。

当前设计分三层：

1. 风格来源
- 负责确认本次视觉路线
- 可以来自内置 `style_preset`
- 也可以来自用户提供的 `style_card_path`
- 如果用户提供风格参考图，可通过 `style_reference_paths` 或 `reference_image_urls` 传给支持参考图的提供商

当前内置预设：

- `realistic_short_drama`：真实短剧感，降低 AI 塑料感
- `dark_underworld_film`：暗黑地府/诡异经营题材
- `character_design_board`：角色设定板/资产可读性
- `cinematic_concept_art`：电影概念图/主视觉

2. 风格卡
- 负责固定 IP 长期稳定的视觉 DNA
- 例如角色锚点、主色、禁用元素、长期风格方向

3. 任务层
- 负责本次创作目标
- 可以是小说内容输入，也可以是互动式逐轮细化
- 例如人物、场景、镜头、情绪、资产类型

组合结果：

- 风格来源先决定真实短剧、暗黑电影、设定板或概念图方向
- 风格卡保证 IP 长期不跑偏
- 任务层保证每次创作有变化和互动空间

## 角色互动创建再推进一层

当前图片 skill 已不只是接收平铺字段，还支持更明确的角色创建结构：

1. `character_profile`
- 用来承载角色长期稳定信息
- 例如身份、外貌、服装、性格、世界观

2. `identity_anchors`
- 用来锁定必须保持稳定的人物识别点
- 例如脸型、发型轮廓、眼型

3. `continuity_rules`
- 用来约束后续资产图不能跑偏
- 例如“后续所有图保持同一人物识别”

4. `asset_target`
- 用来表达这一轮到底要生成什么资产
- 例如英雄立绘、半身海报、表情资产、服装设定图

5. `interaction_state`
- 用来承接当前对话阶段
- 例如已锁定项、待确认项、最新用户方向

这样 agent 可以先做角色设定，再按资产目标出图，而不是每次重写一段大 prompt。

## 角色资产包层

当一个角色已经基本锁定后，图片 skill 现在可以继续往下做一层：

- 同一角色头像
- 半身主视觉
- 全身服装方向图
- 表情或姿态方向图

执行方式：

- 传入一份稳定的 `character_sheet`
- 再传入 `asset_bundle`
- skill 自动展开成多个子任务并统一返回

这样后面 agent 就能先“锁人”，再“一次出一组资产”，更接近真实生产流程。

## IP 资产包层

当输入是一段剧本文案时，不应只生成主角。当前资产拆解标准分为：

1. 多角色
- 每个重要角色独立生成设计图
- 角色设计图默认使用干净纯背景或中性背景
- 不生成剧情场景背景
- 重点展示脸、体型、服装轮廓、材质、身份道具

2. 角色道具
- 与角色身份强绑定的道具优先放在同一张角色设计图中
- 例如老板的菜单账本、红酒杯、手帕；牛头的菜刀、托盘
- 独立世界观道具可作为 `standalone_props` 单独生成

3. 场景图
- 场景不和角色设计图混在一起
- 场景资产优先生成 720 度全景 / 宽幅环境参考
- 提示词必须要求左右边缘无缝衔接
- 建议使用 `21:9` 或符合 GPT Image 2 规格的自定义宽幅尺寸

4. GPT Image 2 规格
- 角色设计图默认 `3:4`、`2K`
- 道具图默认 `1:1`、`2K`
- 720 全景图默认 `21:9`、`4K`
- 自定义尺寸需要遵守 GPT Image 2 的宽高、16 倍数、比例和像素上限规则

## 角色迭代状态层

为了让 agent 多轮对话不丢状态，当前又补了一层确定性状态管理：

- 把每轮用户输入沉淀回 `character_sheet`
- 记录哪些信息已经锁定
- 记录最近一次用户方向
- 记录决策日志
- 自动判断当前还缺哪些关键信息
- 给出下一轮建议提问

这层不依赖外部模型，主要是给 agent 做稳定状态承接。

现在 agent 可以这样工作：

1. 先初始化一份 `character_sheet`
2. 每轮对话把结构化更新写入 `character_state`
3. 从 `summary.next_questions` 里决定下一轮问什么
4. 一旦信息够了，就转成 `character_create` 或 `character_asset_bundle`

## 典型用法

### 1. 小说/文案输入驱动

输入一段小说或文案内容，结合 `asset_kind`、`character_name`、`scene` 等字段，让 skill 生成角色图、角色资产图或场景图。

### 2. 单图互动式创建

通过 `interaction_notes` 或 `conversation_turns` 持续补充要求，让同一个角色图逐轮细化，而不是每次都从零重写一整段 prompt。

### 3. 已有单图继续细化

当已经有一张角色图时，可传入：

- `src_path`
- `interaction_notes`
- `conversation_turns`
- `refine_level`

让 skill 以当前图为基础继续优化，默认保持人物身份、脸部、主体和构图连续。

## 输出规则

所有产物落到 `IMAGE_OUTPUT_ROOT` 指向的目录。

默认：

`E:\Plans for 2026\ip-skills\outputs`

如果未配置环境变量，代码默认回退到项目根目录下的 `outputs`。
